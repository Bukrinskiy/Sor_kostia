СТАНДАРТ ПЕРЕВОДА ЭЛЕКТРОННЫХ СВИДЕТЕЛЬСТВ О РОЖДЕНИИ
АРГЕНТИНА / BUENOS AIRES (CABA)


1. НАЗНАЧЕНИЕ

Данный файл определяет СТРОГИЕ ПРАВИЛА перевода электронных
свидетельств о рождении, выданных Правительством города Буэнос-Айрес
(Gobierno de la Ciudad de Buenos Aires), а также связанных документов:

- электронное свидетельство о рождении (Acta de nacimiento)
- дополнительный лист подписей (Hoja adicional de firmas)
- подтверждение выдачи / загрузки из базы ЗАГС
- апостиль (Apostilla de La Haya)

Переводы предназначены для представления в консульские учреждения
и должны соответствовать уже ПРИНЯТОМУ образцу перевода
электронного свидетельства.


2. ЭТАЛОН ПЕРЕВОДА (КЛЮЧЕВОЕ)

В проекте существует ОДИН ЭТАЛОННЫЙ ПЕРЕВОД,
который считается ПРАВИЛЬНЫМ и ПРИНЯТЫМ для электронного свидетельства.

Этот эталон:

- был принят именно как перевод ЭЛЕКТРОННОГО свидетельства
- содержит корректные формулировки для электронных документов
- содержит правильные названия государственных органов
- корректно разделяет:
  - физическое лицо
  - должностное лицо
  - систему электронной подписи
- не смешивает человека и электронную систему
- не содержит вольных формулировок

ДАННЫЙ ЭТАЛОН ЯВЛЯЕТСЯ ИСТОЧНИКОМ ИСТИНЫ.


3. ОСНОВНОЕ ПРАВИЛО (ОБЯЗАТЕЛЬНО)

ЗАПРЕЩЕНО:
- придумывать формулировки от себя
- перефразировать эталон
- упрощать названия органов
- заменять устойчивые формулировки синонимами

РАЗРЕШЕНО:
- брать формулировки ТОЛЬКО из эталонного перевода
- копировать:
  - заголовки
  - структуру
  - наименования органов
  - формулировки про электронную подпись
  - примечания

ЕСЛИ ФОРМУЛИРОВКА ЕСТЬ В ЭТАЛОНЕ — ИСПОЛЬЗУЙ ТОЛЬКО ЕЁ.


4. ЕСЛИ ЧЕГО-ТО НЕТ В ЭТАЛОНЕ

Если в переводимом документе присутствует элемент,
которого НЕТ в эталонном переводе, допускается:

- сформулировать перевод по аналогии с эталоном
- ОБЯЗАТЕЛЬНО явно указать, что формулировка добавлена не из эталона
- ОБЯЗАТЕЛЬНО задать вопрос о допустимости формулировки

ЗАПРЕЩЕНО:
- молча добавлять новые формулировки
- делать вид, что формулировка «очевидна»


5. ЭЛЕКТРОННЫЙ ХАРАКТЕР ДОКУМЕНТОВ (КРИТИЧЕСКИ ВАЖНО)

Перевод ОБЯЗАН явно указывать, что документ является ЭЛЕКТРОННЫМ.

ОБЯЗАТЕЛЬНО используются формулировки:
- «электронный документ»
- «электронная запись»
- «электронная подпись»

ЗАПРЕЩЕНО:
- писать «подписал» без указания электронной формы
- писать «подпись» без слова «электронная»
- приписывать электронной подписи свойства собственноручной


6. ЭЛЕКТРОННАЯ ПОДПИСЬ

Необходимо ЧЕТКО РАЗДЕЛЯТЬ:

- должностное лицо
- систему электронной подписи
- систему электронного документооборота

Допустимые формулировки берутся ТОЛЬКО из эталона.

ЗАПРЕЩЕНО:
- смешивать ФИО человека и систему подписи
- писать, что человек «подписал документ»,
  если подпись является электронной


7. НАЗВАНИЯ ГОСУДАРСТВЕННЫХ ОРГАНОВ

Названия органов:

- не переводятся свободно
- не сокращаются
- не упрощаются

Используются ТОЛЬКО формулировки из эталона, например:
- «Правительство города Буэнос-Айрес»
- «Реестр гражданского состояния и правоспособности лиц»
- «Главное управление — Реестр гражданского состояния и правоспособности лиц»
- «Система электронного документооборота (GEDO)»


8. СТИЛЬ ПЕРЕВОДА

Перевод должен быть:

- формально-юридическим
- сухим
- нейтральным
- без пояснений «от переводчика»
- без улучшения читаемости
- без интерпретаций

ЦЕЛЬ — ВОСПРОИЗВЕСТИ ДОКУМЕНТ,
А НЕ ОБЪЯСНИТЬ ЕГО.


9. СТРУКТУРА ВЫХОДНОГО ТЕКСТА

Каждый перевод:

- сохраняет структуру оригинала
- не объединяет разные документы
- может быть напрямую:
  - скопирован в Google Docs
  - передан нотариусу
  - использован для консульства


10. ГЛАВНОЕ ПРАВИЛО (РЕЗЮМЕ)

ЕСЛИ ФОРМУЛИРОВКА ЕСТЬ В ЭТАЛОНЕ — БЕРИ ТОЛЬКО ЕЁ.
ЕСЛИ ФОРМУЛИРОВКИ НЕТ — ОСТАНОВИСЬ И УТОЧНИ.
